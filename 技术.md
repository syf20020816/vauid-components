# Vauid Components 技术规划文档

## 1. 组件体系总览

一个健全的音视频组件库应包含以下层次的组件：

```text
┌─────────────────────────────────────────────────────────────┐
│                        业务层组件                             │
│  MeetingRoom / Classroom / LiveStream / RemoteSupport       │
└───────────────────────────────┬─────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────┐
│                        功能层组件                             │
│  ControlBar / ParticipantList / ChatPanel / ScreenShare     │
└───────────────────────────────┬─────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────┐
│                        核心层组件                             │
│  Layout / Tile / Whiteboard / AudioLevel / VideoPreview     │
└───────────────────────────────┬─────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────┐
│                        基础层组件                             │
│  Button / Toggle / Slider / Tooltip / Modal / Dropdown      │
└─────────────────────────────────────────────────────────────┘
```

## 2. 核心层组件

### 2.1 Layout（虚拟布局容器）

**功能**：Grid / Focus / Fullscreen 布局模式，DOM 恒定 + Transform 驱动

**技术要点**：
- 所有 Tile 节点在 DOM 中始终存在，通过 `transform` 控制视觉位置
- 非当前页 Tile 设置 `opacity: 0` + `pointer-events: none`，不销毁 DOM
- 布局计算移至 Web Worker，避免阻塞主线程
- 智能末尾填补算法：删除实体时用末尾实体填补，Transform 计算量 O(1)
- `ResizeObserver` 监听容器尺寸，100ms 防抖重算
- LRU 缓存相同尺寸/人数/布局模式的计算结果

**扩展方向**：
- 支持自定义布局模式（如 Sidebar、Custom Grid）
- 支持拖拽排序，用户手动调整实体位置
- 支持键盘导航（Tab/方向键切换焦点）
- 弱网自适应：检测到弱网时自动降级布局（Grid → Focus）

---

### 2.2 Tile（媒体渲染单元）

**功能**：统一的媒体渲染容器，支持 Video / Audio / Iframe / Canvas 等多种内容类型

**子类型**：

| 子类型 | 用途 | 关键技术 |
|--------|------|----------|
| `Tile.Video` | 视频流渲染 | WebRTC track 绑定、0 帧解码控制、视频质量指示器 |
| `Tile.Audio` | 纯音频渲染 | AudioContext 管理、音量可视化、静音检测 |
| `Tile.Iframe` | 网页/白板嵌入 | 沙箱隔离、跨域通信、状态保持 |
| `Tile.Canvas` | 自定义绘制 | WebGL/2D 渲染、离屏 Canvas、帧率控制 |

**技术要点**：

**Tile.Video**：
- 使用 `ref` 直接操作 `<video>` 元素，避免 React 重渲染
- 非可视区域执行 `video.pause()` 停止解码，保留 WebRTC audio track
- 使用 `requestVideoFrameCallback` 精确控制帧渲染
- 支持 `playsinline`、`autoplay`、`muted` 属性自动管理
- 视频黑屏/卡顿检测：定时检查 `video.readyState` 和帧更新

**Tile.Audio**：
- 使用 `AudioContext.createMediaStreamSource()` 绑定 WebRTC track
- 使用 `AnalyserNode` 实现音量可视化（`getByteFrequencyData`）
- 静音检测：连续 N 帧音量低于阈值触发 `onSilent` 回调
- 支持音频输出设备切换（`HTMLMediaElement.setSinkId`）

**Tile.Iframe**：
- 使用 `sandbox` 属性限制权限（`allow-scripts allow-same-origin`）
- 通过 `postMessage` 实现跨域通信
- DOM 恒定架构下 iframe 不会重载，内部状态 100% 保留
- 支持懒加载：非可视区域 iframe 延迟加载或暂停内容更新

**Tile.Canvas**：
- 支持 WebGL 和 2D 两种渲染上下文
- 非可视区域降低帧率或暂停渲染（`cancelAnimationFrame`）
- 支持离屏 Canvas 预渲染，提升性能
- 可接入 Three.js / PixiJS 等第三方渲染引擎

**扩展方向**：
- 支持自定义渲染器（通过 `renderer` prop 注入）
- 支持媒体流切换（热切换 track，不重建 DOM）
- 支持画中画模式（Picture-in-Picture API）
- 支持截图/录制功能（`canvas.captureStream` / `MediaRecorder`）

---

### 2.3 Whiteboard（协作白板）

**功能**：多人实时协作白板，支持画笔、形状、文字、图片等

**技术要点**：
- 使用 Canvas 或 SVG 作为渲染层
- 操作指令通过 WebSocket 广播，使用 CRDT 或 Operational Transform 解决冲突
- 支持撤销/重做（命令模式 + 操作日志）
- 支持图层管理（背景层、绘制层、标注层）
- 支持导出为 PNG / PDF / SVG
- 触控笔压感支持（`PointerEvent.pressure`）

**扩展方向**：
- 支持 LaTeX 公式渲染
- 支持代码块（Monaco Editor 嵌入）
- 支持 AI 辅助（手写识别、图形自动校正）
- 支持版本历史和时间轴回放

---

### 2.4 AudioLevel（音频电平指示器）

**功能**：实时显示音频音量水平，用于麦克风/扬声器状态指示

**技术要点**：
- 使用 `AnalyserNode` 获取频域数据
- 使用 `requestAnimationFrame` 驱动 UI 更新
- 支持多种显示模式：条形图、圆形、波形
- 支持阈值告警（音量过高/过低/静音）

**扩展方向**：
- 支持多通道音频（立体声分离显示）
- 支持频谱分析（FFT 可视化）
- 支持音频录制波形预览

---

### 2.5 VideoPreview（视频预览）

**功能**：本地摄像头预览，用于入会前设备调试

**技术要点**：
- 使用 `navigator.mediaDevices.getUserMedia()` 获取本地流
- 支持镜像模式（`transform: scaleX(-1)`）
- 支持分辨率/帧率切换
- 支持虚拟背景（WebGL + 人体分割）

**扩展方向**：
- 支持多摄像头切换
- 支持美颜滤镜（磨皮、美白、锐化）
- 支持 AR 特效（贴纸、虚拟形象）

## 3. 功能层组件

### 3.1 ControlBar（控制栏）

**功能**：会议控制工具栏，包含麦克风、摄像头、屏幕共享、离开会议等按钮

**组件清单**：

| 组件 | 功能 |
|------|------|
| `MicToggle` | 麦克风开关，显示静音状态 |
| `CameraToggle` | 摄像头开关，显示视频状态 |
| `ScreenShareButton` | 屏幕共享按钮 |
| `LeaveButton` | 离开/结束会议 |
| `ParticipantCount` | 参会人数显示 |
| `MeetingTimer` | 会议时长计时 |
| `NetworkQuality` | 网络质量指示器 |
| `MoreMenu` | 更多操作菜单 |

**技术要点**：
- 使用 `navigator.mediaDevices.enumerateDevices()` 枚举设备列表
- 使用 `MediaStreamTrack.getSettings()` 获取当前设备状态
- 屏幕共享使用 `getDisplayMedia()`，支持窗口/标签页/全屏选择
- 网络质量通过 WebRTC Stats API（`getStats()`）获取
- 支持快捷键（M 静音、V 视频、S 共享、Q 离开）

**扩展方向**：
- 支持自定义按钮顺序和显隐
- 支持主题定制（颜色、图标、圆角）
- 支持浮动模式（可拖拽移动）
- 支持手势控制（移动端滑动调节音量/亮度）

---

### 3.2 ParticipantList（参会者列表）

**功能**：显示所有参会者，支持静音管理、角色设置、踢出等

**技术要点**：
- 虚拟滚动（`react-window` 或自定义实现），支持大量参会者
- 支持搜索/筛选（按名称、角色、状态）
- 支持拖拽排序（主持人置顶、按发言排序）
- 支持批量操作（全体静音、取消静音）

**扩展方向**：
- 支持举手功能
- 支持分组讨论（Breakout Rooms）
- 支持参会者信息卡片（头像、昵称、角色标签）

---

### 3.3 ChatPanel（聊天面板）

**功能**：会议内文字聊天，支持表情、图片、文件传输

**技术要点**：
- 虚拟列表渲染，支持海量消息
- 消息去重（基于消息 ID 或时间戳）
- 支持消息撤回、编辑、删除
- 支持 @提及 和消息回复
- 文件传输使用 WebRTC DataChannel 或 HTTP 上传

**扩展方向**：
- 支持消息翻译
- 支持消息搜索
- 支持聊天记录导出
- 支持 AI 摘要（会议要点自动提取）

---

### 3.4 ScreenShare（屏幕共享视图）

**功能**：展示共享的屏幕/窗口/标签页内容

**技术要点**：
- 使用 `Tile.Iframe` 或 `Tile.Video` 渲染共享内容
- 支持共享源切换（屏幕 → 窗口 → 标签页）
- 支持共享者标注（共享者名称、共享类型图标）
- 支持共享内容录制

**扩展方向**：
- 支持远程标注（在共享画面上画圈标注）
- 支持共享内容放大/缩小
- 支持多共享源同时展示（分屏模式）

---

## 4. 交互层组件

### 4.1 MouseMapper（鼠标映射层）

**功能**：在远程协作场景中，将本地鼠标操作映射到远程端

**技术要点**：
- 使用 `PointerEvent` 捕获鼠标位置和动作
- 坐标转换：本地坐标 → 远程坐标（考虑分辨率、缩放比例差异）
- 使用 WebSocket 或 DataChannel 传输鼠标事件
- 支持鼠标样式自定义（远程端显示本地鼠标光标）
- 支持右键菜单、滚轮、拖拽等完整鼠标行为

**扩展方向**：
- 支持触控手势映射
- 支持键盘事件映射
- 支持延迟补偿（预测鼠标位置）

---

### 4.2 AnnotationLayer（标注层）

**功能**：在视频/共享画面上叠加标注（画笔、箭头、文字、高亮）

**技术要点**：
- 使用绝对定位的 Canvas 覆盖在内容上方
- 标注数据使用 JSON 格式存储，支持实时同步
- 支持标注图层管理（多人标注时分图层显示）
- 支持标注撤销/清除
- 使用 `pointer-events: none` 穿透标注层操作底层内容

**扩展方向**：
- 支持标注回放
- 支持标注导出为图片
- 支持 AI 辅助标注（自动框选、智能高亮）

---

### 4.3 EffectsLayer（特效层）

**功能**：视频特效处理，包括虚拟背景、美颜、滤镜等

**技术要点**：
- 使用 WebGL 进行视频帧处理
- 虚拟背景：使用 MediaPipe Selfie Segmentation 或 TensorFlow.js BodyPix
- 美颜：使用卷积滤波（磨皮）、色彩调整（美白）
- 滤镜：使用 CSS `filter` 或 WebGL Shader
- 支持特效预览和实时切换

**扩展方向**：
- 支持 AR 贴纸（人脸关键点检测 + 贴图）
- 支持虚拟形象（Avatar）
- 支持绿幕抠图（色度键控）

---

## 5. 基础层组件

基础层组件建议使用成熟的 UI 组件库（如 Radix UI、Headless UI）作为基础，进行二次封装以适配音视频场景。

| 组件 | 用途 |
|------|------|
| `Button` | 通用按钮，支持图标 + 文字、加载状态 |
| `Toggle` | 开关按钮，用于麦克风/摄像头状态切换 |
| `Slider` | 滑块，用于音量/亮度调节 |
| `Tooltip` | 工具提示，用于按钮说明 |
| `Modal` | 模态框，用于设置面板、确认对话框 |
| `Dropdown` | 下拉菜单，用于设备选择、更多操作 |
| `Badge` | 徽章，用于状态指示（静音、共享中） |
| `Avatar` | 头像，用于参会者展示 |

## 7. 通用技术注意事项

### 7.1 性能优化

- **避免不必要的重渲染**：使用 `React.memo`、`useMemo`、`useCallback` 优化组件
- **直接 DOM 操作**：对于高频更新（如视频帧、音频电平），使用 `ref` 直接操作 DOM，避免 React 状态更新
- **Web Worker 卸载计算**：布局计算、图像处理等耗时操作移至 Worker
- **虚拟列表**：长列表使用虚拟滚动，只渲染可视区域节点
- **0 帧渲染**：非可视区域暂停视频解码、降低 Canvas 帧率

### 7.2 内存管理

- **及时释放资源**：组件卸载时清理 `MediaStream`、`AudioContext`、`WebGL Context`
- **避免内存泄漏**：移除事件监听器、取消定时器、清理缓存
- **对象池**：频繁创建/销毁的对象（如标注数据）使用对象池复用

### 7.3 兼容性

- **浏览器兼容**：Chrome / Firefox / Safari / Edge 最新两个版本
- **移动端适配**：iOS Safari / Android Chrome 触控手势适配
- **WebRTC 兼容**：处理不同浏览器的 WebRTC API 差异
- **降级策略**：不支持的特性优雅降级（如 `setSinkId` 不支持时隐藏音频输出选择）

### 7.4 无障碍

- **键盘导航**：所有交互元素支持键盘操作
- **屏幕阅读器**：提供 `aria-*` 属性，状态变化时通知屏幕阅读器
- **高对比度模式**：支持系统高对比度主题
- **字体缩放**：支持浏览器字体大小调整

### 7.5 安全

- **CSP 策略**：支持 Content Security Policy，限制 iframe 来源
- **沙箱隔离**：iframe 使用 `sandbox` 属性限制权限
- **XSS 防护**：用户输入内容转义后渲染
- **权限管理**：麦克风/摄像头/屏幕共享权限请求和状态管理

### 7.6 主题定制

- **CSS 变量**：使用 CSS Custom Properties 实现主题切换
- **Design Token**：定义统一的设计令牌（颜色、间距、圆角、字体）
- **暗色模式**：支持系统暗色模式自动切换
- **品牌定制**：支持自定义 Logo、主色、字体

## 8. 扩展机制

### 8.1 插件系统

```typescript
// 插件接口
interface Plugin {
  name: string;
  install(engine: Engine): void;
  uninstall(engine: Engine): void;
}

// 使用示例
engine.use(new WeakNetworkPlugin());
engine.use(new AnalyticsPlugin());
```

### 8.2 组件组合

```tsx
// 自定义布局
<Layout
  nodes={nodes}
  renderTile={(node) => (
    <Tile.Video
      stream={streams[node.entityId]}
      paused={node.hidden}
    />
  )}
/>

// 自定义控制栏
<ControlBar>
  <MicToggle />
  <CameraToggle />
  <CustomButton /> {/* 自定义按钮 */}
  <ScreenShareButton />
  <LeaveButton />
</ControlBar>
```

### 8.3 事件系统

```typescript
// 监听引擎事件
engine.on('onUpdate', () => { /* 布局更新 */ });
engine.on('onResize', (w, h) => { /* 尺寸变化 */ });
engine.on('onEntityUpdate', (type) => { /* 实体变化 */ });

// 监听媒体事件
tile.on('play', () => { /* 视频播放 */ });
tile.on('pause', () => { /* 视频暂停 */ });
tile.on('error', (err) => { /* 播放错误 */ });
```

### 8.4 渲染器扩展

```tsx
// 自定义渲染器
<Tile
  renderer="custom"
  render={(ctx, frame) => {
    // 自定义 Canvas 渲染逻辑
    ctx.drawImage(frame, 0, 0);
  }}
/>
```

## 9. 开发规范

### 9.1 目录结构

```text
components/
├── {component-name}/
│   ├── index.tsx           # 组件入口
│   ├── {Component}.tsx     # 组件实现
│   ├── {Component}.module.css  # 样式（CSS Modules）
│   ├── hooks/              # 相关 Hooks
│   ├── utils/              # 工具函数
│   ├── types.ts            # 类型定义
│   └── __tests__/          # 测试文件
```

### 9.2 命名规范

- 组件：PascalCase（`TileVideo`、`ControlBar`）
- Hooks：camelCase，以 `use` 开头（`useEngine`、`useMediaStream`）
- 工具函数：camelCase（`computeLayout`、`parseStream`）
- 类型：PascalCase，接口以 `I` 开头或直接使用名词（`LayoutNode`、`IEngineConfig`）
- 常量：UPPER_SNAKE_CASE（`DEFAULT_PAGE_SIZE`、`MAX_CACHE_SIZE`）

### 9.3 导出规范

```typescript
// index.ts - 统一导出
export { Layout } from './Layout';
export { Tile } from './Tile';
export type { LayoutProps, TileProps } from './types';
export { LayoutTypes, DeviceTypes } from './constants';

// 默认导出（仅用于主要组件）
export default Layout;
```

- 集成测试：Playwright
